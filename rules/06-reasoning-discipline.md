# Disciplina de razonamiento

**Hipótesis antes de experimento.** Antes de ejecutar cualquier experimento o análisis
exploratorio, define: (1) una pregunta binaria, (2) criterio de éxito, (3) criterio de
falsificación. Si no puedes definir la falsificación, no es un experimento, es pesca:
no se ejecuta.

**Síntoma vs. causa estructural.** Si un sistema falla repetidamente pese a correcciones
sucesivas, para antes de seguir parcheando. Señales de causa estructural: (a) las mejores
correcciones están al límite de lo que el sistema puede hacer; (b) hay un cambio de
contexto externo que coincide con el inicio del fallo; (c) la asunción fundamental del
sistema ya no se cumple. Acción: re-examinar las asunciones, no añadir más ajustes.

**Documentar el razonamiento de cierre.** Cuando se archiva una dirección, se rechaza una
feature o se abandona una rama de investigación: documenta la evidencia que llevó a la
decisión y qué tendría que cambiar para reconsiderarla. Nunca borres sin dejar constancia
del "por qué no" — la evidencia negativa tiene valor; sin documentar, la próxima sesión
vuelve a explorar el callejón sin salida.
